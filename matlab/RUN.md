Once you run the MATLAB script, you should expect three things:

1. MATLAB prints the numerical results in the Command Window.
2. MATLAB creates weights_matlab.csv in your data/ folder.
3. MATLAB opens two plots.

1. Command Window output

You should see something like this:

MMSE Wiener Filter Lab - MATLAB Verification
-------------------------------------------
Autocorrelation matrix R:
    0.92...    0.01...   -0.02...
    0.01...    0.92...    0.01...
   -0.02...    0.01...    0.92...
Cross-correlation vector P:
    0.63...
   -0.36...
    0.18...
Estimated Wiener weights from MATLAB:
    0.699...
   -0.399...
    0.199...
MSE from actual error:      0.0024...
MMSE formula Ed2 - P^T W:   0.0024...

The exact numbers depend on the random data generated by Python, but the important part is this:

Estimated Wiener weights from MATLAB:
approximately [0.7, -0.4, 0.2]

Because your Python data generator created the desired signal using this hidden system:

h = [0.7; -0.4; 0.2]

So MATLAB should recover weights close to that.

2. weights_matlab.csv should be created

After running the script, this file should appear:

data/weights_matlab.csv

It should contain something like:

0.699...
-0.399...
0.199...

Then your Python verification script can compare:

weights_python.csv
weights_c.csv
weights_matlab.csv

Ideally, Python, C, and MATLAB should be almost identical, with tiny numerical differences only.

3. First plot: desired signal vs filter output

The first MATLAB figure compares:

d(n) = desired signal
y(n) = Wiener filter output

You should expect the two curves to almost overlap.

That means the Wiener filter successfully learned the mapping from the input vector:

[x(n), x(n-1), x(n-2)]

to the desired signal.

Because the desired signal includes noise, the curves will not be perfectly identical, but they should track each other closely.

4. Second plot: estimation error

The second figure shows:

e(n) = d(n) - y(n)

You should expect this error signal to be small and centered around zero.

It will look like low-amplitude random noise. That is expected because the filter can learn the deterministic part:

0.7x(n) - 0.4x(n-1) + 0.2x(n-2)

but it cannot perfectly predict the added random noise:

v(n)

So the remaining error is mostly the noise component.

The key validation

The most important check is this:

MSE from actual error ≈ MMSE formula Ed2 - P^T W

These two values should be almost the same.

That proves both sides of the theory are consistent:

Actual measured error:

mean(e.^2)

Theoretical MMSE formula:

E[d^2(n)] - P^T Wopt

So if MATLAB prints something like:

MSE from actual error:      0.00243791
MMSE formula Ed2 - P^T W:   0.00243791

then your implementation is working correctly.

Small improvement I recommend

After:

W_matlab = R \ P;

add this:

disp('Condition number of R:');
disp(cond(R));

If cond(R) is small or moderate, your matrix is healthy. If it is extremely large, the Wiener solution may become numerically sensitive.

For this Version 1 project, you should expect the full story to be:

MATLAB loads R and P from Python
MATLAB solves R W = P
MATLAB finds weights close to [0.7, -0.4, 0.2]
MATLAB saves weights_matlab.csv
MATLAB plots d(n), y(n), and e(n)
Python can then compare MATLAB, Python, and C results
